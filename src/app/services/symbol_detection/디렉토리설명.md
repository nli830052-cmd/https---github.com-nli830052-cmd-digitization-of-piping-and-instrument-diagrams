# Symbol Detection 서비스 디렉토리 설명 및 학습 가이드

이 폴더(`src/app/services/symbol_detection`)는 P&ID 도면 이미지에서 밸브, 펌프 등 각종 기계 기호(심볼)를 AI 모델을 통해 인식하는 핵심 로직을 담당합니다.

---

## 📂 폴더 및 파일 구조 설명

- **`symbol_detection_service.py` (Core)**
    - 이 서비스의 **컨트롤러(두뇌)** 역할을 합니다.
    - 주요 로직: AI 모델 요청(`endpoint_client` 호출), 도면의 특정 영역(Bounding Box) 필터링, 인식 결과의 임계값(Threshold) 비교, 중복 제거 로직 호출, 결과 이미지 생성.
- **`symbol_detection_endpoint_client.py`**
    - AI 모델이 구동되는 외부 엔드포인트(예: Azure ML)와 통신을 담당하는 **통신 도구**입니다.
    - HTTP 요청 형식, 인증 토큰 처리, 에러 핸들링 로직이 들어 있습니다.
- **`utils/`**
    - **`symbol_pruning_utils.py`**: AI가 한 심볼을 여러 번 중복으로 인식했을 때, 겹치는 박스들을 하나로 정리하는 'Pruning(가지치기)' 알고리즘을 구현합니다.
- **`input/`** (임시 폴더)
    - 로컬 테스트 환경 등에서 인식할 대상 이미지를 넣어두는 용도로 사용됩니다.
- **`output/`** (결과 폴더)
    - 로컬 테스트 결과인 인식 박스가 그려진 이미지(`.png`)와 결과 데이터(`.json`)가 생성되는 경로입니다.

---

## 🧭 권장 학습 순서

효율적으로 코드를 파악하려면 **"이미지가 어떻게 시스템을 거쳐 나가는지"** 그 흐름을 따라가는 것이 좋습니다.

1.  **`symbol_detection_service.py`** (전체 흐름 파악)  
    - 먼저 서비스 전체 시나리오를 읽어보며 큰 틀을 잡으세요.
2.  **`symbol_detection_endpoint_client.py`** (데이터 통신 구조)  
    - 어떻게 AI 서버에 데이터를 보내고 받는지 세부 방식을 분석하세요.
3.  **`utils/symbol_pruning_utils.py`** (세부 알고리즘)  
    - 비즈니스 로직에서 결과 데이터를 어떻게 가공하고 정제하는지 확인하세요.
