# P&ID Digitization 프로젝트 공부 목차

이 프로젝트는 P&ID(배관 및 계장도) 도면 이미지를 디지털 데이터(그래프 데이터베이스)로 변환하는 AI 기반 웹 API 서비스입니다. 효율적인 학습을 위해 아래 순서로 코드를 분석하는 것을 추천합니다.

---

## 📅 학습 로드맵 및 단계별 가이드

### 1단계: 프로젝트 개요 및 설정 (Overview & Configuration)
프로젝트의 전체적인 설계 의도와 시스템 환경을 파악하는 단계입니다.
- **[README.md](file:///c:/digitization-of-piping-and-instrument-diagrams/src/README.md)**: 프로젝트 목적, API 엔드포인트 목록, 주요 알고리즘 설정값(Config) 파라미터가 상세히 기재되어 있습니다.
- **[init_app.py](file:///c:/digitization-of-piping-and-instrument-diagrams/src/init_app.py)**: FastAPI 애플리케이션의 시작점입니다. 서버가 어떻게 초기화되고 미들웨어가 설정되는지 확인하세요.
- **[app/config.py](file:///c:/digitization-of-piping-and-instrument-diagrams/src/app/config.py)**: AI 모델의 임계값(Threshold), 스토리지 경로명 등 전역 설정값을 관리합니다.

### 2단계: API 인터페이스 (Interface Layer)
외부에서 시스템 기능을 호출하는 진입점을 분석합니다.
- **[app/routes/controllers/pid_digitization_controller.py](file:///c:/digitization-of-piping-and-instrument-diagrams/src/app/routes/controllers/pid_digitization_controller.py)**: 핵심 비즈니스 로직이 각 API 엔드포인트에 어떻게 연결되는지 보여줍니다.
    - `detect_symbols`: 심볼 감지 (Post)
    - `detect_text`: 텍스트 인식 및 심볼과 연계 (Post)
    - `detect_lines_and_construct_graph`: 라인 감지 및 전체 그래프 생성 (Post)

### 3단계: 핵심 비즈니스 로직 (Service Layer)
이 프로젝트의 핵심인 `src/app/services/` 폴더 내 도메인 로직을 순서대로 분석합니다.
1.  **symbol_detection**: 이미지에서 밸브, 펌프 등 객체를 인식하는 로직.
2.  **text_detection**: OCR(Azure Form Recognizer 등)을 사용해 글자를 읽고 근처 심볼과 속성을 매칭.
3.  **line_detection**: 도면의 선(배관)을 추적하고 추출하는 알고리즘(OpenCV 활용).
4.  **graph_construction**: 인식된 심볼과 라인을 결합해 논리적인 연결 관계를 도출하는 단계.
5.  **graph_persistence**: 최종 그래프 구조를 데이터베이스에 저장하는 과정.

### 4단계: 데이터 모델링 (Data Model Layer)
시스템 내부에서 정보가 교환될 때 사용되는 규격을 확인합니다.
- **app/models/**: Bounding Box(영역), 심볼 정보, 라인 정보 등에 대한 데이터 클래스(Pydantic/POJO) 정의.
- **graph-db-model/**: 그래프 데이터베이스(SQL Graph 등)에 저장되는 데이터의 스키마와 관계 정의.

### 5단계: 기타 및 인프라 (Utils & Infrastructure)
- **app/utils/**: 좌표 변환, 이미지 처리(OpenCV) 등 범용적으로 사용되는 함수 모음.
- **app/services/blob_storage_client.py**: Azure Blob Storage와 연동하여 이미지 및 중간 결과물을 저장하는 로직.
- **_tests_/**: 각 기능이 정상 작동하는지 확인하기 위한 유닛 테스트 코드.
