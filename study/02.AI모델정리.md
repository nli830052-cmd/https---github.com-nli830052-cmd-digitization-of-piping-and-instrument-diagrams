# P&ID Digitization 프로젝트 사용 AI 모델 분석

이 프로젝트는 여러 단계의 AI 모델과 컴퓨터 비전 알고리즘을 결합한 **멀티-스테이지 파이프라인(Multi-stage Pipeline)** 방식을 채택하고 있습니다. 전체적인 기술 스택과 각 단계별 작동 방식은 다음과 같습니다.

---

## 1. 이미지 처리 및 인식 단계 (Recognition Layer)

### 1-1. 심볼 인식 (Symbol Detection)
도면 이미지 내에서 밸브, 펌프, 계기류 등의 특정 객체 위치와 종류를 찾아냅니다.
- **모델**: Custom Object Detection 모델 (YOLO, Faster R-CNN 계열 등)
- **플랫폼**: **Azure Machine Learning (AML)** 엔드포인트
- **동작 방식**: 이미지 바이트 데이터를 클라우드 엔드포인트로 전송(`REST API`)하고, 각 심볼의 **Bounding Box(위치 좌표)**와 **Label(종류)**, **Confidence Score(확률)**를 반환받습니다.

### 1-2. 텍스트 인식 (Text Detection / OCR)
도면에 적힌 문자(태그 번호, 수치, 설명 등)를 읽어옵니다.
- **모델**: **Azure AI Document Intelligence (구 Form Recognizer)**
- **세부 모델**: `prebuilt-read` 고해상도 모드
- **특징**: `OCR_HIGH_RESOLUTION` 기능을 사용하여 도면 특유의 아주 작은 글씨나 선과 겹쳐진 텍스트를 정밀하게 추출합니다. 인식된 텍스트는 좌표 단위로 변환되어 이후 심볼과의 관계 분석에 사용됩니다.

---

## 2. 구조적 분석 단계 (Structural Analysis Layer)

### 2-1. 라인 감지 (Line Detection)
도면에서 배관(Pipe)에 해당하는 선들의 좌표를 실시간으로 계산하여 추출합니다.
- **기술**: **전통적 컴퓨터 비전 (Traditional Computer Vision)**
- **라이브러리**: **OpenCV** (Python `cv2`)
- **주요 알고리즘**:
    - **Adaptive Thresholding**: 도면의 밝기 불균형에 대응하여 흑백으로 변환.
    - **Thinning (Skeletonization)**: 굵은 배관 선을 1픽셀 두께의 선으로 얇게 만들어 좌표 추출을 용이하게 함.
    - **Hough Transform (허프 변환)**: 픽셀 데이터를 기반으로 도면 내의 직선 경로를 수학적으로 계산하여 추출.

### 2-2. 그래프 구성 (Graph Construction)
인식된 점(심볼)과 선(배관)들을 논리적으로 연결하여 시스템의 전체 구성도를 만듭니다.
- **기술**: **기하학적 추론 및 그래프 알고리즘**
- **주요 라이브러리**:
    - **Shapely**: 기하학적 연산(두 선이 만나는지, 점이 선 위에 있는지 등 계산).
    - **NetworkX**: 점과 선의 연결 관계를 시스템의 그래프 구조로 관리.
- **동작 방식**: "A 배관"의 끝 점이 "B 펌프"의 영역(Bounding Box) 주변에 위치하는지 기하학적으로 검증하여 연결 고리를 형성합니다.

## 💡 기술 전략 요약

| 구분 | 적용 기술 | 장점 |
| :--- | :--- | :--- |
| **인식 (Recognition)** | 클라우드 기반 딥러닝 (Azure) | 대용량 모델을 사용하여 인식 정확도와 유연성을 극대화함. |
| **분석 (Structural)** | 로컬 알고리즘 (OpenCV / Math) | 도면의 정밀한 좌표 데이터를 기반으로 시스템 연결의 정확성을 확보함. |

이 프로젝트는 모든 분석을 AI에만 맡기지 않고, **"AI의 인식력"**과 **"전통 알고리즘의 정밀성"**을 영리하게 결합하여 도면 디지털화의 신뢰도를 높이고 있습니다.
